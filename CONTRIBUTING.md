# Contributing to Motion-Lyte-JS

First off, thank you for considering contributing to Motion-Lyte-JS! It's people like you that make the open-source community such an amazing place.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub issue tracker.
- Check if the bug has already been reported.
- Provide a clear, concise description and steps to reproduce.

### Suggesting Enhancements
- Open an issue with the "enhancement" label.
- Explain why this feature would be useful.

### Pull Requests
1. Fork the repo.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes.
4. Push to the branch.
5. Open a Pull Request.

## Development Setup
1. Clone the repository.
2. Run `npm install`.
3. Use `npm run dev` to start the development environment.
4. Run `npm test` before submitting your PR.

## Style Guide
- Follow the existing code style (ESLint configured).
- Use JSDoc for all new functions and types.
- Ensure accessibility standards are met.
